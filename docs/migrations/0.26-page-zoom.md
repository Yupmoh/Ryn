# Migrating to 0.26.0: main-window page zoom

Ryn 0.26.0 adds `IRynWindow.SetPageZoom` / `GetPageZoom` and the matching
`window.setPageZoom` / `window.getPageZoom` commands.

```js
await window.__ryn.invoke('window.setPageZoom', { factor: 0.8 });
```

Page zoom is clamped to `0.25–5.0`. Ryn uses native engine zoom where available and keeps DOM-derived
native coordinate bridges aligned:

- `window.setTitleBarDragRegions` remains raw page CSS pixels; Ryn scales drag and ignore rectangles.
- `webviewPane.open` and `webviewPane.setBounds` remain page CSS pixels; Ryn scales pane bounds before
  native placement and re-applies them when page zoom changes.

Hosts that previously reached the native webview themselves should replace that code with
`window.setPageZoom`. Remove any bridge wrapper that multiplies `window.setTitleBarDragRegions` arrays;
manual rect scaling would double-scale coordinates after the upgrade. A wrapper installed after Ryn's
injected title-bar script is harmlessly bypassed because the script captures the original invoke function,
but it should still be deleted.

Pane-content zoom is unchanged and remains independent through `webviewPane.setZoom`.
