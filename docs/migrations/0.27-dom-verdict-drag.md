# 0.27: DOM-verdict window dragging

0.27 replaces the published-rectangles drag model (page streams `data-webview-drag`/`data-webview-ignore`
bounding boxes to a native hit-testing overlay) with a live-DOM verdict at mousedown time. The injected
script decides per click whether the point is draggable; on macOS it posts `window.beginNativeDrag` and the
host starts the drag from the **retained original mousedown NSEvent** (`performWindowDragWithEvent:`), so
IPC latency cannot desync the window from the cursor. No geometry is published anymore — nothing goes
stale, and overlays/popovers are handled by construction.

## Removed

- `IRynWindow.SetTitleBarDragRegions` and the `window.setTitleBarDragRegions` command. There are no
  regions to publish; delete any direct calls. Explicit layout publication is replaced by the attributes
  plus `RynOptions.TitleBarAutoDragHeight`.
- `RynOptions.TitleBarDragView`. There is no native drag view. The event monitor is passive (it only
  retains the last mousedown between down and up) and only ever acts on a page verdict, so an app that
  self-manages dragging simply doesn't use the attributes.

## Added

- `RynOptions.TitleBarAutoDragHeight` (also on `RynWindowOptions`, and bindable from `Ryn:` config):
  height in CSS pixels of an automatic top drag strip — a zero-markup native-feeling title bar.
- `IRynWindow.BeginNativeDrag(x, y)` / `window.beginNativeDrag`: macOS drag from the retained mousedown,
  verified against the verdict point. Driven by the injected script; apps rarely call it directly.

## Behavior changes

- Interactive elements inside a `data-webview-drag` region are now excluded **automatically** (semantic
  elements, ARIA widget roles, `[onclick]`, `[draggable="true"]`). `data-webview-ignore` is only needed
  for non-semantic clickables (a bare `div` with a JS listener).
- **Stop marking layout wrappers `data-webview-ignore`** — it excludes the wrapper's whole rectangle,
  including empty space, from dragging. Mark individual controls instead (or nothing, if semantic).
- Double-click-to-zoom now applies to any draggable point on all platforms (previously native-only on
  macOS drag regions).
